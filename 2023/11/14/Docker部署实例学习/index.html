Docker部署案例学习

一、🐬mysql容器部署

要求

能用的5.7版本的mysql容器

细化要求

  1. 使用的镜像是mysql:5.7
  2. 后台运行 -d
  3. 数据持久化存储 -v
  4. 开放3306端口 -p
  5. 设置root密码 -e
  6. 停止后自动重启 –restart
  7. 容器命名为 rbac_mysql

命令编写

docker run -d \
-v mysql_data:/var/lib/mysql \
-p 3306:3306 \
-e MYSQL_ROOT_PASSWORD=root \
--name rbac_mysql \
--restart always \
mysql:5.7

可以使用Navicat连接数据库建库建表

二、🎞️redis容器部署

要求

能用的7版本redis

细化

  1. redis 7
  2. 后台运行 -d
  3. 数据持久化存储 -v
  4. 开放6379端口 -p
  5. 停止后自动重启 –restart
  6. 容器命名为 rbac_redis
  7. 开启redis时，开启aof持久化 redis-server –appendonly yes

命令编写

docker run \
-d \
-v redis_data:/data \
-p 6379:6379 \
--restart always \
--name rbac_redis \
redis:7.0 redis-server --appendonly yes

三、🍀后端应用部署

要求

使用openjdk 8u111版木，去运行springboot服务，保证该服务能正常使用java -jar ssadmin-server-2.7.17.jar

server.port=xxxx

细化

  1. 镜像java:openjdk-8u111
  2. 开放端口 7777
  3. 后台运行 -d
  4. 数据卷同步jar包到容器 -v
  5. 停止后自动重启 –restart
  6. 容器命名为 rbac_admin

命令编写

docker run \
-p 7777:7777 \
-d \
-v /usr/rbac:/usr/rbac \
--restart always \
--name rbac_admin \
java:openjdk-8u111 java -jar ./usr/rbac/ssadmin-server-2.7.17.jar

测试接口

curl --location --request POST 'http://localhost:7777/user/login' \
--header 'Content-Type: application/json' \
--data '{
     "username":"zjz",
     "password":"1234"
}'

发现无法连通，这是因为后端容器和mysql容器还不在一个网络内，无法相互通信

目标：将需要互相通信的容器加入到一个网络内，用容器名代替ip

#创建网络
docker network create 网络名
#查看网络

#加入网络
#1.创建容器时加入 
docker run --network 网络名 镜像名      
#2.创建后加入 docker network connect rbac_net rbac_admin
docker network connect [选项] 网路名 容器名|容器id  
#删除网络
docker network rm 网络名或id

五、🐕‍🦺容器网络连通

创建网络

docker network create rbac_net

将容器加入网络

docker network connect rbac_net rbac_mysql
docker network connect rbac_net rbac_redis

后端应用容器在创建时加入

先删除原来的后端应用容器

docker rm -f 容器名或ID
docker run \
-p 7777:7777 \
-d \
--network rbac_net \
-v /usr/rbac:/usr/rbac \
--restart always \
--name rbac_admin \
java:openjdk-8u111 java -jar ./usr/rbac/ssadmin-server-2.7.17.jar \
"--spring.datasource.url=jdbc:mysql://rbac_mysql:3306/rbac?characterEncoding=utf-8&serverTimezone=UTC" \
"--spring.datasource.username=root" \
"--spring.datasource.password=root" \
"--spring.redis.host=rbac_redis" \

再次测试登录接口

curl --location --request POST 'http://localhost:7777/user/login' \
--header 'Content-Type: application/json' \

六、👗部署前端应用

要求

使月nginx1.21.5版本部署sg-blog-vue让用户能正常访问

细化

  1. 镜像nginx:1.21.5
  2. 开放端口 80
  3. 数据卷方式同步静态资源 -v
  4. 停止后自动重启 –restart
  5. 后台运行 -d
  6. 容器命名为 rbac_vue

编写命令

docker run -d \
-p 80:80 \
-v /usr/rbac/rbac_vue/dist:/usr/share/nginx/html \
--restart always \
--name rbac_vue \
nginx:1.21.5

不用加入网络，但是要将baseURL修改为宿主机的ip

浏览器输入服务器（宿主机）ip……
