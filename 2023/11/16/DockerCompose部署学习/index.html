DockerCompose部署学习

一、为什么用DockerCompose？😏

之前学习docker到DockerFile发现部署变得简洁了，但是要部署多个容器时还是要一个一个运行，麻烦

DockerCompose用来定义一个或多个容器运行和应用

当前版本docker带有dockercompose不用单独安装

输入docker-compose version有版本输出说明已安装

二、入门🏋🏻‍♀️🏋🏻

编写dockercompose文件

docker-compose.yml

services:
  rbac:
    image: rbac:01

rbac:01是你当前拥有的 镜像名:tag

运行命令

docker—compose up -d

看到容器开始运行了，rbac:01是后端应用

docker ps 查看容器运行情况

停止运行

docker-compose down

三、一些命令项㊙️

四、部署实例🧻🧻

编写docker-compose.yml

services:
  #数据库服务
  rbac_mysql:
    image: mysql:5.7
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - 3306:3306
    environment:
      MYSQL_ROOT_PASSWORD: root
    restart: always
    networks:
      - rbac_net
  #redis服务
  rbac_redis:
    image: redis:7.0
    volumes:
      - redis_data:/data 
    ports:
      - 6379:6379
    restart: always
    command: ['redis-server','--appendonly','yes']
    networks:
      - rbac_net
  #后端服务
  rbac_admin:
    image: rbac_admin:01
    ports:
      - 28080:28080
    networks:
      - rbac_net
    restart: always
  #前端服务
  rbac_vue:
    image: rbac_vue:01
    ports:
      - 80:80
    restart: always
networks:
  rbac_net:
volumes:
  mysql_data:
  redis_data:

当然前后端应用的镜像是我们自定义的，因此需要自己先在服务器上构建好

参考：DockerFile部署学习

后端：

FROM openjdk:8u111

# 指定工作目录
WORKDIR /app

# 添加jar包到镜像中
ADD ssadmin-server-2.7.17.jar /app/

# 声明开放端口
EXPOSE 28080

# 启动命令
CMD ["java", "-jar", "ssadmin-server-2.7.17.jar", "--spring.datasource.url=jdbc:mysql://rbac_mysql:3306/rbac?characterEncoding=utf-8&serverTimezone=UTC", "--spring.datasource.username=root", "--spring.datasource.password=root"]

build一下

docker build -t rbac:01 -f rbac_dockerfile .

前端

FROM nginx:1.21.5
WORKDIR /app
COPY rbac_vue .
RUN cp -r dist/* /usr/share/nginx/html
EXPOSE 80
ENTRYPOINT ["nginx","-g","daemon off;"]

同样build一下

docker build -t rbac_vue:01 -f rbac_vue_dockerfile .

最后，dockercompose登场

docker-compose up -d

然后就

docker ps 查看

完事！不想用就down一下

docker-compose down
